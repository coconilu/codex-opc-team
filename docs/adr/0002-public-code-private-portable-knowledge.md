# ADR-0002：公开代码与私人可移植知识分离

- **Status:** Accepted
- **Date:** 2026-07-13

## Context

OPC 需要长期保存经理偏好、项目历史、决策、经验和流程。这些知识高度私人，但插件代码、Schema 和角色机制需要公开、可安装和可贡献。如果把知识放在插件目录或公开仓库，升级会覆盖数据，卸载可能误删数据，也会产生严重隐私风险。

知识同时不能被某个运行时或供应商锁定，否则“一起成长”会在迁移时中断。

## Decision

公开仓库只包含行为代码、Schema、空白模板、测试和脱敏合成示例。用户真实知识存放在独立、用户拥有、默认私有的 File/Git 仓库，通过 `OPC_KNOWLEDGE_HOME` 或安装器文档化的用户级路径发现。

插件缓存、运行日志和可选索引都不属于权威知识。卸载/升级插件不得删除或覆盖私人知识。用户可以在不使用 Codex OPC Team 的情况下直接读取、导出和版本控制知识。

## Consequences

### Positive

- 开源仓库可以安全分享；
- 用户拥有数据和 Git 历史，便于审计、备份和迁移；
- 插件版本与知识生命周期解耦；
- 可以替换记忆/召回实现。

### Negative

- 安装器需要处理路径发现、Schema 版本和迁移；
- 用户可能需要理解插件、运行数据和知识目录的区别；
- 多设备同步由用户的私人 Git/备份策略承担。

## Rejected alternatives

- **知识直接放插件目录：** 缓存和升级生命周期不可靠。
- **知识放公开项目仓库：** 隐私边界错误，并会污染业务项目历史。
- **只放 Mem0/数据库：** 不透明、难审计且产生后端锁定。
- **默认上传托管服务：** 超出 v0.1 数据授权和运维边界。
