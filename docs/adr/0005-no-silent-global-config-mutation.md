# ADR-0005：不静默修改 Codex 全局配置

- **Status:** Accepted
- **Date:** 2026-07-13

## Context

本地原型通过个人 `config.toml` 注册命名角色，并将配置路径硬编码到本机插件源码。这能快速验证流程，却无法安全分发：不同用户配置可能有其他用途，共享 Feature Flag 可能被多个插件依赖，安装器无法在没有所有权信息时判断卸载应删除什么。

全局配置是用户环境的一部分，错误修改可能影响所有 Codex 项目。

## Decision

OPC 核心通过 Plugin Skills 和 Codex 通用子 Agent 能力工作，不要求静默修改用户全局配置。

任何可选角色注册或配置增强必须执行：读取现状、生成精确 Diff、标识插件拥有的键、向用户说明影响并取得明确确认、备份原文件、原子写入、验证和提供恢复方式。

卸载只移除可证明由本插件创建、且未被用户修改的配置项。共享 Feature Flag 不因卸载 OPC 被盲目关闭。配置不得引用维护者本机绝对路径。

## Consequences

### Positive

- 安装不会意外破坏其他 Codex 工作流；
- 开源分发不依赖作者环境；
- 变更可审计和回滚；
- 核心体验在最小权限下可用。

### Negative

- 某些命名角色体验需要额外的用户确认步骤；
- 安装器需要可靠的 Diff、备份和所有权记录；
- 插件需要适配 Codex 通用委派而非依赖全局角色名。

## Rejected alternatives

- **安装时直接覆盖 `config.toml`：** 不可接受的数据和可用性风险。
- **要求用户手工复制固定配置：** 易错、不可迁移且无法判断所有权。
- **卸载时关闭所有相关 Feature：** 可能破坏其他插件和用户设置。
- **继续硬编码插件源码路径：** 只在单机原型有效，无法从缓存/Marketplace 安装。
