# Capability evolution candidate boundary

A retrospective may recommend a role, Skill, or organization-policy destination, but it only creates a source candidate. It must not create a candidate commit, edit the capability, authorize a pilot, or claim promotion.

Pass the portable candidate ID plus hashed feedback/evaluation/lineage refs to the curator. The later capability proposal must add exact current/candidate/rollback Git versions, scope, owner, independent QA, manager approvals, paired pilot results, and a rollback path. Keep observation separate from causal inference and state `association/evidence only`.
