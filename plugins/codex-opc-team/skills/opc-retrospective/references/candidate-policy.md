# Experience Candidate Policy

Create a candidate only when every gate passes:

| Gate | Requirement |
|---|---|
| Provenance | Cite a resolvable run or artifact with a relative canonical reference |
| Reuse | State scope and trigger beyond the one incident |
| Causality | Separate observation from inference |
| Action | Name a concrete future behavior change |
| Safety | Exclude secrets, sensitive traits, raw payloads, and absolute paths |
| Testability | Name replay, shadow, regression, or equivalent evidence |

Prefer “When X, do Y because Z; verify with V.” Keep raw evidence in its project or evaluation boundary, not in the reusable lesson.
