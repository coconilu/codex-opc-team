# Hierarchical context handoff

Use `opc_hierarchical.py` only after resolving exact `project_id`, role, sensitivity authorization, canonical knowledge root, and private memory data root. Treat `ContextPacket` L2 citations as usable context; L0/L1 and `RecallTrace` are diagnostics, never facts.

If the trace reports flat fallback, continue safely and disclose degradation only when it matters to the task. Never rebuild during query. Index build/delete require a separate zero-write preview and manager approval of the unchanged token. A packet or evaluation result cannot approve, obsolete, merge, commit, reindex, or publish knowledge.
