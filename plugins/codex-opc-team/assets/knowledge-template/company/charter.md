# 公司章程

## 组织目的

把 Codex 作为统一工作台，让经理负责方向、范围、审美和重大风险，让 Agent 团队负责梳理、实现、测试、返工和证据整理。

## 权责

| 主体 | 权责 |
|---|---|
| 经理 | 产品方向、范围、不可逆操作、外部发布、组织知识晋升 |
| 总管 | 理解目标、组队、任务契约、协调、验收汇总、升级风险 |
| 员工 Agent | 在岗位和任务契约内交付产物与证据 |
| QA | 独立核验，不接受开发者自证完成 |
| 经验管理员 | 管理候选、验证、冲突、晋升和回滚，不自动扩权 |

## 完成定义

只有同时具备实现证据、当前验证证据、独立 QA 证据和可体验入口，才允许总管通知经理“可以体验”。

## 组织连续性

员工不是永久进程。员工由版本化岗位、Skill、权限、组织知识和历史评估复原；任何单一线程、模型输出或召回索引都不是公司事实源。
